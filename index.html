结合NG项目(chain-web)进行开发

  • 目前，整个商家后台原版是基于AngularJS进行开发，现在新的业务需求已经基于React框架进行开发，因此在本地开发过程存在需要同时起两个工程项目的 HTTP 服务 进行页面联调的情况
    • chain-web 启动HTTP服务：grunt server 监听9000端口
    • merchants 启动HTTP服务：tnpm start 监听8000端口
  • 页面登陆逻辑在chain-web工程中进行，登陆后会把服务端返回的用户信息（如：用户名、用户ID 等等）存储到localstorage里面，然后 merchants工程 会去获取本地的用户信息进行展示

  • 由于本地域名跨端口无法共享localstorage信息，因此需要通过统一域名+端口访问两个工程页面

    • 此处引入nginx服务，进行代理转发,相关概念逻辑可参考此文档

    • 代理转发配置：

      http {
  server {
      # 本地访问入口（nginx服务）
      listen 7888;
      server_name localhost;

      location / {
          index            index.html index.php;
          proxy_pass      http://localhost:9000;
      }

      location ~* /(style|views) {
          proxy_pass      http://localhost:9000;
      }

       location ~* /(settled|common|inventory) {
          proxy_pass      http://localhost:8000;
      }
  }
}

Markdown 是什么

Markdown是一种轻量级标记语言，创始人为约翰·格鲁伯（英语：John Gruber）。它允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档”。这种语言吸收了很多在电子邮件中已有的纯文本标记的特性。 ——维基百科

为什么用Markdown？

  • 简约: 用最简的语法达到效果
  • 专注: 实现形式与内容的分离
  • 直观: 符合人直觉的表达方式
  • 跨平台：支持的平台越来越多
  • 工具链丰富：编辑器or转换格式
  • 代码友好度高：支持代码直接展示

语法 - 兼容 HTML

Markdown的语法很少，只对应html的一小部分。Markdown只涵盖纯文本可以涵盖的范围。不在markdown涵盖之内的的标签，都可以直接在文档里面用HTML书写。直接加标签就可以。

语法 - 特殊字符自动转换

在 HTML 文件中，有两个字符需要特殊处理： < 和 &，< 符号用于起始标签，& 符号则用于标记 HTML 实体，如果你只是想要显示这些字符的原型，你必须要使用实体的形式，像是 &lt; 和 &amp;。

Markdown 让你可以自然地书写字符，需要转换的由它来处理好了。

比如：AT&T 而不用写成 AT&amp;T
比如：4 < 5 而不用写 4 &lt; 5
